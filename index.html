1
Create and configure your Microsoft Form
▼

Go to forms.microsoft.com and create a new form. Design it with structured question types that will translate cleanly to data columns in Power BI.

✓Use Choice questions for categorical data (department, region, satisfaction level)
✓Use Rating questions (1–5 or 1–10 scales) for KPI calculations
✓Use Date fields if tracking over time
✓Avoid open-text fields for your main metrics — they can't be aggregated easily
✓Enable Response Notifications so you know when new data arrives
💡

Name each question clearly (e.g., "Department", "Satisfaction_Score") — these become your column headers in the data source. Avoid spaces and special characters if possible.

2
Connect responses to SharePoint via Power Automate
▼

The most reliable live connection method is piping each Form submission into a SharePoint List, which Power BI can refresh automatically.

In Power Automate (flow.microsoft.com):

✓Create a new flow: Automated → When a new response is submitted (Microsoft Forms)
✓Add action: Get response details — select your Form ID and Response ID
✓Add action: Create item (SharePoint) — map each form field to a SharePoint column
✓Add a Timestamp column using utcNow() expression for time intelligence in Power BI
// Power Automate expression for submission timestamp utcNow() // Alternatively, convert to local timezone convertTimeZone(utcNow(),'UTC','Asia/Manila')
⚠️

Make sure the SharePoint site column types match the form data types (Number columns for ratings, Date columns for dates, Single line of text for everything else).

3
Connect Power BI to SharePoint (OData or SharePoint connector)
▼

Open Power BI Desktop and connect directly to your SharePoint List:

✓Home → Get Data → SharePoint Online List
✓Paste the SharePoint site root URL (not the list URL): https://yourorg.sharepoint.com/sites/yoursite
✓Select 2.0 implementation for better performance
✓Navigate to your list → Transform Data to open Power Query

In Power Query Editor, clean the data:

// Remove unnecessary SharePoint system columns // Keep only: ID, Title, your form fields, Timestamp // Change column types correctly Table.TransformColumnTypes(Source, { {"Satisfaction_Score", Int64.Type}, {"Submission_Date", type datetime}, {"Department", type text} })
4
Build a Date table for time intelligence
▼

Time intelligence DAX functions require a proper Date table. Create one in the Data view:

// Create Date Table in Power BI (New Table) Date_Table = ADDCOLUMNS( CALENDAR( DATE(2024,1,1), DATE(2026,12,31) ), "Year", YEAR([Date]), "Month", MONTH([Date]), "MonthName",FORMAT([Date],"MMMM"), "Quarter", QUARTER([Date]), "Weekday", WEEKDAY([Date],2), "IsWeekend",IF(WEEKDAY([Date],2)>=6,1,0) )

Then mark it as a Date Table: Table tools → Mark as date table → Date column. Create a relationship from Date_Table[Date] to your form responses' Submission_Date column.

5
Write DAX measures for KPI cards
▼

Create these core measures in the Modeling tab → New Measure:

// Total responses Total_Responses = COUNTROWS(FormResponses) // Average satisfaction score Avg_Score = AVERAGE(FormResponses[Satisfaction_Score]) // Responses in selected period vs prior period Responses_Prior = CALCULATE([Total_Responses], DATEADD(Date_Table[Date], -1, MONTH) ) // MoM growth % MoM_Growth = DIVIDE( [Total_Responses] - [Responses_Prior], [Responses_Prior], 0 ) // % of low scores (score below 3) Low_Score_Pct = DIVIDE( COUNTROWS( FILTER(FormResponses, FormResponses[Satisfaction_Score] < 3) ), [Total_Responses] ) // Completion rate (where all required fields filled) Completion_Rate = DIVIDE( COUNTROWS(FILTER(FormResponses, FormResponses[Department] <> BLANK() && FormResponses[Satisfaction_Score] <> BLANK() )), [Total_Responses] )
6
Build the dashboard layout — filters, charts, KPI cards
▼

Lay out your report page in this order (top → bottom):

✓Row 1 — Header bar: Title text box + company logo image + last refresh timestamp using NOW()
✓Row 2 — Slicer bar: Date range slicer (between style) + Department slicer (dropdown) + Region slicer
✓Row 3 — KPI cards: 4 cards using your DAX measures (Total Responses, Avg Score, Completion Rate, Low Scores)
✓Row 4 — Charts: Line chart (responses over time) + Clustered bar (responses by department) + Donut (score distribution)
✓Row 5 (optional): Table visual showing latest 20 responses with conditional formatting on score column
💡

Enable Edit interactions under Format to control which visuals filter which others. Set the date slicer to filter all, but let the department bar chart cross-filter only the line chart, not the KPI cards (to keep totals visible).

// KPI Card — show colored trend arrow // In Format pane of Card visual: // Data label → Conditional formatting → Icon rule // Rule: if [MoM_Growth] > 0 → green up arrow // Rule: if [MoM_Growth] < 0 → red down arrow // Conditional formatting on table score column: // Background color rules: // Score 1-2 → Red | Score 3 → Amber | Score 4-5 → Green
7
Set up automatic scheduled refresh in Power BI Service
▼

Publish your report and configure live refresh:

✓Home → Publish → select your workspace in Power BI Service
✓In Service: Dataset → Settings → Scheduled Refresh
✓Set refresh frequency: up to 48x/day on Pro, 1x/day on free
✓For near-real-time: use DirectQuery mode in Power BI Desktop for the SharePoint connection (rechecks every 15 min automatically)
⚠️

DirectQuery mode disables some DAX functions like time intelligence. Use Import mode with frequent scheduled refresh as a balance for most survey dashboards.

8
Export as PDF & PowerPoint automatically
▼

📄 Manual Export

  1. File → Export → Export to PDF
  2. File → Export → Export to PowerPoint
  3. Exports the current filtered view
  4. Each report page = one slide / PDF page

⚡ Automated via Power Automate

  1. Create a scheduled flow (e.g., every Monday 7 AM)
  2. Action: Export to file for Power BI reports
  3. Set format: PDF or PPTX
  4. Action: Send via email or save to SharePoint
// Power Automate HTTP action body for PDF export { "format": "PDF", "powerBIReportExportConfiguration": { "settings": { "locale": "en-US" }, "pages": [ { "pageName": "ReportSection1" } ] } }
💡

You can pass slicer values into the export API to send department-specific PDFs to each department head automatically — a powerful personalized reporting pattern.

// OEE calculation — the SCADA gold standard metric // OEE = Availability × Performance × Quality Availability = DIVIDE( SUM(Production[RunTime_min]), SUM(Production[RunTime_min]) + SUM(Production[DownTime_min]) ) Performance = DIVIDE( SUM(Production[UnitsProduced]), SUMX(Production, RELATED(Machines[NominalSpeed]) * Production[RunTime_min] / 60 ) ) Quality = DIVIDE( SUM(Production[UnitsProduced]) - SUM(Production[UnitsRejected]), SUM(Production[UnitsProduced]) ) OEE = [Availability] * [Performance] * [Quality]
3
Alarm management DAX and visuals
▼
// Active alarms = acknowledged but not cleared Active_Alarms = COUNTROWS( FILTER(Alarm_Log, Alarm_Log[AcknowledgedTime] <> BLANK() && Alarm_Log[ClearedTime] = BLANK() ) ) // Unacknowledged alarms (highest priority) Unack_Alarms = COUNTROWS( FILTER(Alarm_Log, Alarm_Log[AcknowledgedTime] = BLANK()) ) // Mean Time to Acknowledge (MTTA) in minutes MTTA = AVERAGEX( FILTER(Alarm_Log, Alarm_Log[AcknowledgedTime] <> BLANK()), DATEDIFF( Alarm_Log[AlarmTime], Alarm_Log[AcknowledgedTime], MINUTE ) ) // Mean Time to Resolve (MTTR) MTTR = AVERAGEX( FILTER(Alarm_Log, Alarm_Log[ClearedTime] <> BLANK()), DATEDIFF( Alarm_Log[AlarmTime], Alarm_Log[ClearedTime], MINUTE ) )
💡

For the alarm list visual, use a Table with conditional formatting: Critical = red background, Warning = amber, Info = blue. Sort by AlarmTime descending so newest alarms appear first, just like a real SCADA HMI.

Core maintenance DAX measures
▼
// Equipment Availability % Equipment_Availability = DIVIDE( (24 * 30) - SUM(Downtime[DowntimeHours]), 24 * 30 ) // MTBF — Mean Time Between Failures (hours) MTBF = DIVIDE( SUM(Downtime[OperatingHours]), COUNTROWS(FILTER(WorkOrders, WorkOrders[WOType] = "Corrective")) ) // MTTR — Mean Time to Repair (hours) MTTR = AVERAGEX( FILTER(WorkOrders, WorkOrders[Status] = "Closed"), DATEDIFF(WorkOrders[StartDate], WorkOrders[CompletionDate], HOUR) ) // PM Compliance % — completed on time PM_Compliance = DIVIDE( COUNTROWS( FILTER(WorkOrders, WorkOrders[WOType] = "Preventive" && WorkOrders[CompletionDate] <= WorkOrders[DueDate] ) ), COUNTROWS(FILTER(WorkOrders, WorkOrders[WOType] = "Preventive")) ) // Total Maintenance Cost Total_Maint_Cost = SUMX(WorkOrders, WorkOrders[PartsCost] + (WorkOrders[LaborHours] * 500) // ₱500/hr labor rate ) // Upcoming PMs in next 7 days (alert flag) Upcoming_PMs_7d = COUNTROWS( FILTER(PM_Schedule, PM_Schedule[NextDueDate] >= TODAY() && PM_Schedule[NextDueDate] <= TODAY() + 7 ) )

4
Downtime trend and Pareto analysis
▼

Build a Pareto chart to identify which failure codes cause the most downtime — the 80/20 rule is always visible in maintenance data:

✓Bar chart: Downtime hours by Failure Code (sorted descending)
✓Line overlay: Cumulative % of downtime (secondary Y axis)
✓Area/Stacked bar: Monthly downtime split by Planned vs Unplanned
✓Scatter chart: MTBF vs MTTR per equipment — quickly identifies worst performers
// Cumulative % for Pareto line (create in DAX) Cumulative_Downtime_Pct = VAR CurrentCode = SELECTEDVALUE(Downtime[DowntimeCause]) VAR TotalDT = CALCULATE(SUM(Downtime[DowntimeHours]), ALL(Downtime[DowntimeCause])) VAR CumulativeDT = CALCULATE( SUM(Downtime[DowntimeHours]), FILTER( ALL(Downtime[DowntimeCause]), CALCULATE(SUM(Downtime[DowntimeHours])) >= CALCULATE(SUM(Downtime[DowntimeHours]), FILTER(ALL(Downtime), Downtime[DowntimeCause] = CurrentCode) ) ) ) RETURN DIVIDE(CumulativeDT, TotalDT)
5
Automated alerts with Power Automate
▼

Set up data-driven alerts so the team is notified before problems escalate:

Alert Type 1 — Data Alert in Service

In Power BI Service, click a KPI card → click the bell icon → set threshold. Example: alert when Unplanned Downtime hours > 8 this week. Sends email automatically.

Alert Type 2 — Power Automate Flow

Trigger: Power BI data alert triggered. Action: Post to Teams channel "Maintenance Alerts" with equipment name, downtime hours, and suggested WO number.

Alert Type 3 — PM Due Reminder

Scheduled flow daily at 7 AM. Query SharePoint PM list for NextDueDate within 3 days. Send Teams message to assigned technician with equipment details.

Alert Type 4 — Weekly Report Email

Every Monday morning: export maintenance dashboard PDF → send to maintenance manager with summary of the prior week's KPIs pre-populated.
